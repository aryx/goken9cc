This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Goken9cc is a portable multi-platform C compiler, assembler, and linker toolchain
rooted in Ken Thompson's Plan 9 operating system. Forked from the Go repository
(October 2010), which itself came from inferno-os and plan9port. Written in C89/C99.

## Build Commands

The build system uses Plan 9 `mk` (not GNU Make). The Makefile is a thin wrapper.

```bash
# First-time setup
./configure                    # generates mkconfig (detects arch/OS)
./scripts/build-mk.sh          # bootstrap mk and rc from POSIX shell
./scripts/promote-mk.sh        # copy mk/rc binaries to ./bin/
source env.sh                  # set PATH, MKSHELL, NPROC

# Build
make                           # delegates to mk
make install                   # install to ROOT/$arch/bin/ and ROOT/$arch/lib/

# Test
make test                      # runs all tests via mk test (tests/mkfile)

# Docker (works on both amd64 and arm64)
make build-docker              # build toolchain
make build-docker-test         # build + run regression tests (includes QEMU)
make build-docker-principia    # build + compile Principia OS kernels
```

## Architecture Naming Convention

Plan 9 uses single-character codes for architectures. Each tool is prefixed with this code:

| Code | Arch | Compiler | Assembler | Linker | Object ext |
|------|------|----------|-----------|--------|------------|
| 5 | arm | 5c | 5a | 5l | .5 |
| 7 | arm64 | 7c | 7a | 7l | .7 |
| 8 | x86 (386) | 8c | 8a | 8l | .8 |
| 6 | amd64 | 6c | 6a | 6l | .6 |
| v | mips | vc | va | vl | .v |
| i | riscv | ic | ia | il | .i |

Pipeline: `.c` → compiler (`Xc`) → assembler (`Xa`) → linker (`Xl`) → `X.out`

## Code Organization

- **compilers/** — per-arch C compilers + generic `cc/` frontend and `pcc/` portable wrapper
- **assemblers/** — per-arch assemblers
- **linkers/** — per-arch linkers + `ar` archiver
- **lib_core/** — `lib9/` (Plan 9 POSIX adaptation), `libbio/` (buffered I/O), `libc/` (minimal C lib with OS/ and ARCH/ branches)
- **lib_strings/** — `libregexp/`, `libstring/`, `libflate/`
- **lib_toolchain/** — `libmach/` (object file/binary format parsing, used by `ar`, `acid`, and the emulators — not just debuggers)
- **include/** — master headers (`u.h`, `libc.h`, `bio.h`, `mach.h`), plus `ARCH/` and `OS/` subdirs
- **machines/** — `5i` (ARM emulator), `vi` (MIPS emulator)
- **debuggers/** — `acid` (Plan 9 debugger)
- **generators/** — `lex`, `yacc`
- **mk/** — Plan 9 mk build tool (bootstrapped from shell)
- **rc/** — Plan 9 rc shell
- **GO/** — original Go-era source (historical reference)
- **tests/** — `c/` (mini, mini2, vlong, float), `s/` (assembly), `lib9/`, `go/`, `rc/`
- **src/** — Go-style wrappers (`src/cmd/`) around toolchain components

## Key Headers

- `u.h` — must be included first (POSIX normalization, stdint, etc.)
- `libc.h` — Plan 9 C library declarations
- `bio.h` — buffered I/O
- `mach.h` — binary format/debugging

## Build System Details

- `mkfile` files use Plan 9 mk syntax (not Makefile syntax)
- `mkconfig` — generated by `./configure`, sets `objtype`, `GOOS`, `GOARCH`
- `mkfiles/mkfile.proto` — shared parameters (CFLAGS, LDFLAGS)
- `mkfiles/$objtype/mkfile` — per-architecture config
- `mkfiles/mklib`, `mkfiles/mkone`, `mkfiles/mkmany` — build templates for libraries/binaries
- Compiler flags: `-O0 -fno-inline -ggdb` (debug-friendly by default)

## Environment Variables

- `GOARCH` / `GOOS` — target architecture/OS (build time)
- `GOROOT` — used for `#9/...` paths (runtime)
- `MKSHELL` — path to rc shell for mk
- `RCMAIN` — rc init file path
- `YACCPAR` — yacc template file path
